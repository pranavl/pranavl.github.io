2015.01.19
Moverio glasses were not found at desk. Box was found in another desk, but device was missing.

When continuing to work with ArUco, it was found that:
  • CPU problems shown in Dependency Walker were because depends.exe was 64-bit
  • 32-bit version of Dependency Walker solved errors, but did not isolate problem
  • Even though JArUco is a Java wrapper, it relies on the native C++ code, which will be very difficult to implement on the smart glasses

As a result, alternate libraries were researched:
Metaio SDK
  • Commonly used and well documented library, but difficult to implement
  • Built using gradle, which is seemingly more common than Ant, with Android
  • Android Studio is good for UI development
DroidAR
  • Not well documented, but easy to load and use
  • Designed specifically for use with Android; buildable in Netbeans

Continued to have problems with ADB USB driver installation.
Was able to circumvent problem to install apps on Moverio glasses:
  1. On computer, navigate to project directory /bin
  2. Locate project_name.apk file
  3. Upload .apk file to online repository (such as box.com)
  4. On Moverio glasses, open browser and navigate to online repo
  5. Download .apk file to install app
Set up git repository for app.
